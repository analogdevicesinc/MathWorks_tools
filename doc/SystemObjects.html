ADI BSP System Objects For Radio I/O

All ADI streaming interfaces share the same general API. Simply insert you specific part of development board:
rx = adi.<part_or_board_name>.Rx;
rx.uri = '<uri of board>';
receiveData = rx();
tx = adi.<part_or_board_name>.Tx;
tx.uri = '<uri of board>';
tx(receiveData);
Example:
rx = adi.AD9361.Rx
rx =
adi.AD9361.Rx with properties:
CenterFrequency: 2.4000e+09
SamplingRate: 3000000
RFBandwidth: 3000000
GainControlModeChannel0: 'slow_attack'
EnableQuadratureTracking: true
EnableRFDCTracking: true
EnableBasebandDCTracking: true
SamplesPerFrame: 32768
channelCount: 2
uri: 'ip:10.0.0.200'
rx.uri = 'ip:192.168.2.1';
rx.CenterFrequency = 1e9;
data = rx();

AD936X Dependencies

To use the AD936X system object will require libad9361. This is packaged within the toolbox installer on the GitHub release page. However, since MathWorks' file exchange does not allow libaries files to be posted this dependency is stripped from the installers on FileExchange. Without this library you will get warnings like:
rx()
Error using loadlibrary
Could not find file ad9361-wrapper.h.
Error in loadlibrary
Error in adi.AD9361.Base/setupLibad9361 (line 108)
[~, ~] = loadlibrary(libName, loadlibraryArgs{:});
Error in adi.AD9361.Rx/setupInit (line 255)
setupLibad9361(obj);
Error in adi.common.RxTx/configureChanBuffers (line 156)
setupInit(obj);
Error in matlabshared.libiio.base/setupImpl
To install the library there is an installer built into the toolbox. Run the following:
dl = adi.utils.Downloader;
dl.download('libad9361');
This will add the necessary pieces to the toolbox install and add them to path.
